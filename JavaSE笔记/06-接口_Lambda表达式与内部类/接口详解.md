# 接口详解：只给规则，不给实现

接口就是“行为规范”。你要实现它，就必须把方法补齐。

---

## 1. 接口是啥（大白话）

- 关键词：`interface`
- 作用：定义“能做什么”
- 实现：`implements`

---

## 2. 接口里的东西

- 抽象方法（默认 public abstract）
- 常量（默认 public static final）
- Java 8+ 还允许 `default` 方法

---

## 3. 接口的好处

- 解决“多继承”需求
- 解耦：面向接口编程

---

## 4. 面试高频问答

### Q1：接口和抽象类区别？
**答：**
接口是“能做什么”，抽象类是“你是什么”。接口可多实现。

### Q2：接口可以有方法体吗？
**答：**
Java 8+ 可以用 `default` 方法。

---

## 5. 一句话总结

接口是“规则清单”，实现类负责“填空答题”。
