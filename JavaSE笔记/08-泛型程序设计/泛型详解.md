# 泛型详解：给类型加“占位符”

泛型就是“先占位，后确定类型”，避免强转和类型错误。

---

## 1. 泛型是啥（大白话）

```java
List<String> list = new ArrayList<>();
```

意思是：这个 List 只装 String。

---

## 2. 泛型类 / 泛型方法

```java
class Box<T> {
    T value;
    T get() { return value; }
}

static <T> T pick(T a, T b) { return a; }
```

---

## 3. 上界/下界（面试高频）

- `<? extends T>`：上界，只能读
- `<? super T>`：下界，可以写

口诀：  
**PECS**：Producer 用 extends，Consumer 用 super。

---

## 4. 类型擦除

泛型在编译期生效，运行期会擦掉类型信息。  
所以运行时看不到 `List<String>` 的 String。

---

## 5. 面试高频问答

### Q1：为什么要用泛型？
**答：**
类型安全，少写强转，编译期就能报错。

### Q2：`List<Object>` 和 `List<String>` 是父子关系吗？
**答：**
不是，泛型不具备协变关系。

---

## 6. 一句话总结

泛型让类型更安全，少出错。
