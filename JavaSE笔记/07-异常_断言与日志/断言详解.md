# 断言详解：给程序加“自检”

断言就是“我觉得这里一定成立”，如果不成立就直接报错。

---

## 1. 断言是啥

```java
assert x > 0 : "x 必须大于 0";
```

大白话：  
如果条件不成立，就抛 `AssertionError`。

---

## 2. 开启断言

默认是关闭的，运行时要加：

```bash
java -ea Main
```

---

## 3. 什么时候用

- 开发/测试阶段做自检
- 不建议放在核心业务逻辑里

---

## 4. 面试高频问答

### Q1：断言和异常区别？
**答：**
异常是可预期错误；断言是开发期的自检工具。

### Q2：断言默认开启吗？
**答：**
默认关闭，需要 `-ea` 开启。

---

## 5. 一句话总结

断言是“程序自检”，不是用户错误处理工具。
