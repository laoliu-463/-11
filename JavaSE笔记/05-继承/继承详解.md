# 继承详解：代码复用的“家族关系”

继承就是“子承父业”。子类能直接拿到父类里公开/受保护的成员，从而少写重复代码。

---

## 1. 继承是啥（大白话版）

- 关系：子类 is-a 父类（例如：猫 is-a 动物）
- 关键字：`extends`
- 规则：**单继承**（一个类只能有一个直接父类）

---

## 2. 重点：super 关键字（高频必考）

`super` 表示“父类这边”。常见用法就两类：

### 2.1 调父类构造器

- 写在子类构造器**第一行**  
- 作用：先把父类那部分初始化好

```java
class Person {
    String name;
    Person(String name) {
        this.name = name;
    }
}

class Student extends Person {
    int id;
    Student(String name, int id) {
        super(name); // 先初始化父类
        this.id = id;
    }
}
```

### 2.2 调父类成员（字段/方法）

当子类和父类**同名**时，用 `super` 指明“我要父类的那个”。

```java
class Parent {
    void work() { System.out.println("父类工作"); }
}

class Child extends Parent {
    void work() {
        super.work(); // 先用父类逻辑
        System.out.println("子类扩展");
    }
}
```

### 2.3 this 和 super 的区别（面试常问）

- `this`：当前对象（子类自己）
- `super`：父类那部分

一句话：`this` 找自己，`super` 找爸爸。

---

## 3. 方法重写（Override）

子类对父类方法“重新做一遍”，但名字和参数必须一致。

要点：
- 方法名 + 参数列表一致
- 访问权限**不能更小**
- 建议加 `@Override`，编译器帮你查错

---

## 4. 构造器执行顺序（必考）

创建子类对象时：

1. 先执行父类构造器
2. 再执行子类构造器

原因：子类里包含父类那部分，必须先把父类装好。

---

## 5. Object 是所有类的祖宗

Java 里所有类默认都继承 `Object`，常见方法：
- `toString()`：对象打印
- `equals()` / `hashCode()`：比较和哈希

---

## 6. 面试高频问答（大白话版）

### Q1：继承的好处是什么？
**答：**
复用代码，关系更清晰，少写重复逻辑。

### Q2：为什么构造器里 `super()` 必须第一行？
**答：**
父类先初始化，子类才有“完整的自己”。

### Q3：`this` 和 `super` 的区别？
**答：**
`this` 指当前对象，`super` 指父类那部分。

### Q4：为什么 Java 不支持多继承？
**答：**
避免冲突（比如两个父类都有同名方法，子类不知道用哪个）。

---

## 7. 一句话总结

继承让代码复用更轻松，`super` 让你明确“我要用父类那部分”。
