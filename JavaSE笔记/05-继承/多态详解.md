# 多态详解：同一句话，不同表现

多态就是“同一个父类引用，指向不同子类对象，表现出来的行为不一样”。

---

## 1. 多态是啥（大白话版）

一句话：**变量看类型，行为看对象**。

```java
Animal a = new Dog();
a.eat(); // 实际执行 Dog 的 eat
```

这里 `a` 看起来是 Animal，但真正干活的是 Dog。

---

## 2. 多态的前提

- 有继承关系
- 子类重写父类方法
- 父类引用指向子类对象（向上转型）

---

## 3. 多态的好处

- 代码更灵活
- 面向接口/父类编程
- 扩展方便：加新子类不改旧代码

---

## 4. 向上转型 vs 向下转型

```java
Animal a = new Dog(); // 向上转型，安全
Dog d = (Dog) a;      // 向下转型，要确认真实类型
```

如果类型不对，会 `ClassCastException`，通常用 `instanceof` 先判断。

---

## 5. 面试高频问答（大白话版）

### Q1：多态的核心是什么？
**答：**
父类引用指向子类对象，调用方法时执行子类版本。

### Q2：为什么多态能让代码更灵活？
**答：**
方法参数写父类类型，传哪个子类都行，扩展时不用改老代码。

### Q3：向上转型和向下转型区别？
**答：**
向上转型安全，向下转型可能出错，需判断真实类型。

---

## 6. 一句话总结

多态的本质就是“表面看父类，实际用子类”。
