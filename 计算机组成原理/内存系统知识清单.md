# 内存系统知识清单 📚

> 基于习题复习整理的核心知识点

## 一、存储器分类

### 1.1 按易失性分类

| 类型 | 定义 | 代表 |
|:---|:---|:---|
| 易失性 (Volatile) | 断电丢失数据 | SRAM、DRAM |
| 非易失性 (Non-volatile) | 断电保持数据 | ROM、Flash、EEPROM |

### 1.2 按存储介质分类

```
存储器
├── 半导体存储器
│   ├── 易失性：SRAM、DRAM
│   └── 非易失性：ROM、Flash、EEPROM
├── 磁存储器：硬盘、磁带
└── 光存储器：CD-ROM、DVD、BD（不是半导体！）
```

## 二、SRAM vs DRAM 核心对比

| 特性 | SRAM | DRAM |
|:---:|:---:|:---:|
| 结构 | **6T 触发器** | **1T1C（晶体管+电容）** |
| 速度 | 快 ✓ | 慢 |
| 成本 | 高 | 低 ✓ |
| 集成度 | 低 | 高 ✓ |
| 刷新 | 不需要 ✓ | 需要（电容漏电） |
| 用途 | **Cache** | **主存** |
| 易失性 | 易失 | 易失 |

### 记忆口诀

> **SRAM** = **S**mall（集成度小）+ **S**peedy（快）+ **S**table（稳定不刷新）+ **S**ix（6T）
> 
> **DRAM** = **D**ense（密集/集成度高）+ **D**elay（要刷新有延迟）+ **D**uo（1T+1C）

## 三、ROM 家族

| 类型 | 全称 | 可写性 | 擦除方式 |
|:---|:---|:---|:---|
| ROM | Read-Only Memory | 出厂固化 | 不可擦 |
| PROM | Programmable ROM | 一次编程 | 不可擦（熔丝烧断） |
| EPROM | Erasable PROM | 多次编程 | 紫外线擦除 |
| EEPROM | Electrically EPROM | 多次编程 | 电擦除（按字节） |
| Flash | 闪存 | 多次编程 | 电擦除（按块） |

### Flash 特性

| 特性 | 说明 |
|:---|:---|
| 读速度 | 快（接近RAM） |
| 写速度 | **慢（需先擦后写）** |
| 易失性 | 非易失 ✓ |
| 访问方式 | 随机访问 |
| 用途 | U盘、SSD、嵌入式存储 |

## 四、易错点汇总

| 易错点 | 正确理解 |
|:---|:---|
| RAM 是"随机存取"所以不会丢数据？ | ❌ RAM 是**易失性**的，断电丢失 |
| CD-ROM 是半导体存储器？ | ❌ CD-ROM 是**光学存储器** |
| DRAM 比 SRAM 快？ | ❌ **SRAM 更快**（触发器 vs 电容） |
| DRAM 比 SRAM 贵？ | ❌ **DRAM 更便宜**（结构简单） |
| Flash 读写速度一样？ | ❌ **读快写慢** |
| 6T 结构是 DRAM？ | ❌ **6T 是 SRAM**，DRAM 是 1T1C |

## 五、高频考点速查

| 问法 | 答案 |
|:---|:---|
| 易失性存储器？ | RAM（SRAM、DRAM） |
| 非易失半导体存储器？ | Flash、EEPROM、ROM |
| 6T 结构？ | **SRAM** |
| 1T1C 结构？ | **DRAM** |
| 需要刷新？ | **DRAM** |
| 读快写慢？ | **Flash** |
| 用于 Cache？ | **SRAM** |
| 用于主存？ | **DRAM** |
| 不是半导体存储？ | CD-ROM、DVD（光学存储） |

## 六、存储层次金字塔

```
          ▲ 速度快、容量小、成本高
          │
    ┌─────────────┐
    │   寄存器    │  ← CPU 内部
    └─────────────┘
    ┌─────────────┐
    │   Cache     │  ← SRAM
    └─────────────┘
    ┌─────────────┐
    │    主存     │  ← DRAM
    └─────────────┘
    ┌─────────────┐
    │    外存     │  ← 磁盘、SSD、Flash
    └─────────────┘
          │
          ▼ 速度慢、容量大、成本低
```
