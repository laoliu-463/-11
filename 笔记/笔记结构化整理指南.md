## 一、模式识别

### 1.1 笔记结构模式

**识别到的标准结构：**

```
1. 元数据区（头部）
   - 标题
   - 描述/摘要
   - 最后更新日期
   - 相关链接

2. 目录区
   - 自动生成或手动维护
   - 使用锚点链接

3. 内容区
   - 按逻辑层次组织
   - 使用标准化的章节结构

4. 总结区（可选）
   - 要点总结
   - 相关链接
   - 参考资料
```

### 1.2 内容组织模式

**识别到的内容模式：**

| 模式类型 | 标识 | 用途 | 示例 |
|---------|------|------|------|
| **概念定义** | `**定义：**` | 给出精确的定义 | `**定义：** CPU是中央处理器` |
| **通俗理解** | `**通俗理解：**` | 用比喻解释概念 | `**通俗理解：** CPU就像大脑` |
| **核心要点** | `【要点】` 或 `✅` | 强调重要内容 | `✅ 核心思想：...` |
| **公式** | 代码块或行内 | 数学公式 | `` `公式 = 值1 × 值2` `` |
| **代码示例** | `**Java代码示例：**` | 提供可运行代码 | Java代码块 |
| **对比表** | Markdown表格 | 对比不同概念 | `\| 特性 \| A \| B \|` |
| **流程图** | ASCII或Mermaid | 展示流程 | Mermaid图表 |
| **示例** | `**示例：**` | 具体例子 | 示例说明 |

### 1.3 格式规范模式

**标题层级规范：**
- `#` - 文档标题（唯一）
- `##` - 主要章节（一、二、三...）
- `###` - 子章节（1.1, 1.2...）
- `####` - 小节（1.1.1, 1.1.2...）

**分隔符使用：**
- `---` - 章节分隔
- `***` - 强调分隔（较少使用）

**代码块标识：**
- ` ```java` - Java代码
- ` ```c` - C代码
- ` ``` ` - 普通代码/文本
- ` ```mermaid` - Mermaid图表

## 三、整理规则

### 3.1 标题层级规则

**规则：**
1. 文档标题使用 `#`（唯一）
2. 主要章节使用 `##`，编号用中文：一、二、三...
3. 子章节使用 `###`，编号用数字：1.1, 1.2...
4. 小节使用 `####`，编号用数字：1.1.1, 1.1.2...

**示例：**
```markdown
# 文档标题

## 一、主要章节

### 1.1 子章节

#### 1.1.1 小节
```

### 3.2 内容组织规则

**标准内容块顺序：**
1. **定义** - 精确的概念定义
2. **通俗理解** - 生活化解释
3. **核心要点** - 关键点列表
4. **详细说明** - 深入解释
5. **公式/算法** - 数学公式或算法步骤
6. **代码示例** - 可运行代码
7. **示例/案例** - 具体例子
8. **对比表** - 对比不同概念
9. **应用场景** - 实际应用
10. **总结** - 要点总结

**规则：**
- 每个章节至少包含"定义"和"通俗理解"
- 复杂概念必须包含"代码示例"
- 涉及算法的必须包含"算法步骤"
- 涉及公式的必须包含"公式说明"

### 3.3 代码示例规则

**Java代码示例标准格式：**
```markdown
**Java代码示例：**
```java
// 类名：描述性名称
class ExampleName {
    // 方法注释：说明方法功能
    ReturnType methodName(Parameters) {
        // 实现代码
        // 关键步骤添加注释
        return result;
    }
    
    // 示例方法
    void example() {
        // 示例调用
        // 输出结果
    }
}
```
```

**规则：**
- 类名使用描述性名称
- 方法添加功能注释
- 关键步骤添加行内注释
- 提供示例调用方法
- 代码要能直接运行（或说明需要的前提条件）

1. **识别模式**
   - 阅读笔记，识别结构模式
   - 标记不符合标准的部分

4. **检查完善**
   - 使用检查清单
   - 补充缺失内容
   - 修正格式错误

**可以自动化的部分：**

## 一、CPU结构

### 什么是CPU？

**定义：** CPU（Central Processing Unit，中央处理器）是计算机的核心部件，负责执行程序指令、进行运算、控制各个部件。

**通俗理解：** CPU就像电脑的"大脑"，负责思考和处理信息。CPU内部由控制器、运算器和寄存器组成，就像一个高效的生产线。

**核心要点：**
- 控制器（CU）：指挥中心
- 运算器（ALU）：计算器
- 寄存器组：高速临时仓库

**Java代码示例：**
```java
// CPU结构Java模拟
class CPU {
    // 控制器
    class Controller {
        void control() {
            // 控制逻辑
        }
    }
    
    // 运算器
    class ALU {
        int calculate(int a, int b) {
            return a + b;
        }
    }
    
    void execute() {
        // 执行指令
    }
}
```
