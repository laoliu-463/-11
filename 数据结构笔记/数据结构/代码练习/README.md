## 🚀 快速开始

### 编译和运行

每个子目录都包含：
- `main.c` - 测试程序
- `xxx.h` - 头文件
- `xxx.c` - 实现文件
- `Makefile` - 编译配置
- `README.md` - 说明文档

#### 使用 Makefile（推荐）
```bash
cd 01-顺序表
make          # 编译
make run      # 编译并运行
make clean    # 清理
```

#### 手动编译
```bash
cd 01-顺序表
gcc -g -Wall -fexec-charset=GBK main.c seqlist.c -o seqlist.exe
.\seqlist.exe
```

## 📝 代码规范

### 文件命名
- 头文件：`数据结构名.h`（如 `seqlist.h`）
- 实现文件：`数据结构名.c`（如 `seqlist.c`）
- 测试文件：`main.c`

### 代码风格
- 使用有意义的变量名和函数名
- 添加详细的注释
- 函数功能单一，职责明确
- 包含错误处理和边界检查

### 测试要求
- 测试基本操作
- 测试边界情况
- 测试错误处理
- 输出清晰的测试结果
