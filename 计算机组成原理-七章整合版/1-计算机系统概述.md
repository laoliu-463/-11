## 知识点简要介绍

1. **体系结构与发展**  
   - 体系结构（ISA）决定“能做什么”，组成原理/微体系结构决定“怎么做”。
   - 冯诺依曼五大部件：运算器、控制器、存储器、输入、输出；改进型哈佛通过指令/数据分离减少冲突。
   - 发展从电子管 → 晶体管 → 中/大规模集成 → 微处理器 → 超大规模集成，性能提升逐渐受功耗与并行度制约。

2. **程序执行三阶段**  
   - 编译阶段：预处理→编译→汇编→链接，生成可执行文件。  
   - 加载阶段：OS 创建进程、映射虚拟地址、初始化 PC/SP。  
   - 运行阶段：CPU 取指→译码→执行→访存→写回，栈帧维护函数调用上下文。

3. **性能指标与计算**  
   - 执行时间 = 指令数 × CPI × 时钟周期；性能 = 1 / 执行时间。  
   - Amdahl 定律：`S = 1 / [(1-P) + P/N]`，串行部分决定上限。  
   - 流水线：`T = (k + n - 1) × Δt`，n → ∞ 时加速比约等于级数 k。

4. **性能优化与综合应用**  
   - 多层优化：硬件/微体系结构/存储/编译器/软件/系统。  
   - 常见策略：前递、分支预测、Cache 调优、TLB、DMA、中断优先级、磁盘调度等。  
   - 设计题套路：需求拆解 → 方案设计 → 量化评估 → 风险说明。

5. **复盘要点**  
   - 画清体系结构与存储层次图，熟悉“编译→加载→执行”细节。  
   - 做好性能公式单位转换，能写出 AMAT / Amdahl / 流水线吞吐计算。  
   - 记得 DMA 与 Cache 的一致性处理、分支预测与流水线停顿的解决办法。
